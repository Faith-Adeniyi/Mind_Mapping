# ClockRail / Mind Mapping App — Future Phases

## 1. Purpose of This Document

This document defines the product expansion path beyond the MVP. It describes what should come after the first usable version so the app can grow without losing focus.

The future phases are not required for the first release, but they help establish a long-term product roadmap.

---

## 2. Phase Structure

The roadmap is organized into stages:

- **Phase 1:** MVP core experience
- **Phase 2:** richer visual and editing capabilities
- **Phase 3:** AI-assisted personalization and export
- **Phase 4:** advanced practice, collaboration, and ecosystem features

---

## 3. Phase 2 — Enhanced Visual Control

### Goals
Phase 2 should make the ClockRail map more flexible and more visually expressive.

### Features
- drag-and-drop repositioning of segments
- manual reordering of segment positions
- richer icon libraries
- color themes or semantic color coding
- stronger editing of labels and text blocks
- more refined presentation mode interactions

### Why This Phase Matters
This phase improves user control without changing the core concept. It keeps the visual metaphor but gives users more freedom to shape the map.

---

## 4. Phase 3 — AI Assistance and Smart Generation

### Goals
Phase 3 should introduce intelligent support for transforming raw text into strong visual memory maps.

### Features
- automatic text summarization
- AI keyword extraction
- AI-assisted segmentation
- concept-to-icon suggestions
- concept-to-image generation
- better segment classification by topic
- auto-tuning of the number of segments based on text length

### Why This Phase Matters
This is where the product becomes more powerful and less manual. The app can help the user produce high-quality maps faster with less editing.

---

## 5. Phase 4 — Practice, Review, and Memory Training

### Goals
Phase 4 should improve memory reinforcement and rehearsal workflows.

### Features
- rehearsal mode with hidden labels
- flashcard-style review
- spaced repetition reminders
- blank-recall practice
- timed speech rehearsal
- progress tracking over multiple practice sessions

### Why This Phase Matters
The app is not only for building a map. It should help users internalize the map and improve recall over time.

---

## 6. Phase 5 — Export and Sharing

### Goals
Phase 5 should allow the user to move the map out of the app and use it in real-world presentations or sharing contexts.

### Features
- export as PNG
- export as PDF
- printable sheet layout
- shareable public or private link
- presenter handout version
- image export for slides or documents

### Why This Phase Matters
Export makes the app useful beyond the browser and supports offline presentation workflows.

---

## 7. Phase 6 — Multi-Layout Support

### Goals
This phase expands the visual language of the app beyond the classic clock rail.

### Possible Layouts
- 12-hour clock rail
- 24-hour clock rail
- spiral layout
- linear timeline layout
- hybrid map layout
- topic cluster layout

### Why This Phase Matters
Different content types may work better with different visual arrangements. A broader layout engine increases product flexibility.

---

## 8. Phase 7 — Cloud, Accounts, and Collaboration

### Goals
This phase turns the app into a reusable product platform.

### Features
- user accounts
- cloud save and sync
- project history
- duplicate and version maps
- sharing and collaboration
- team presentations or classroom use
- cross-device access

### Why This Phase Matters
This expands the app from a single-use tool into a long-term platform.

---

## 9. Phase 8 — Advanced Input Sources

### Goals
This phase adds more ways to start a map.

### Features
- file upload
- PDF import
- Word document import
- speech-to-text capture
- pasted transcript import
- live lecture note capture

### Why This Phase Matters
Users should not be limited to manually pasted text. More input sources make the app practical in more contexts.

---

## 10. Phase 9 — Personal Visual Libraries

### Goals
This phase gives users stronger memory anchors based on their own life and environment.

### Features
- upload personal images
- save personal icon sets
- custom visual memory libraries
- reusable associations per user
- favorite object and symbol sets

### Why This Phase Matters
Personalized visuals are usually easier to remember than generic icons.

---

## 11. Phase 10 — Analytics and Learning Insights

### Goals
This phase helps users improve how they use the app.

### Features
- map creation history
- practice completion tracking
- recall confidence indicators
- segment usage frequency
- preparation time insights
- improvement trends over time

### Why This Phase Matters
Analytics can help users understand how effective their memory maps are and where they may need more practice.

---

## 12. Recommended Future Priorities

The suggested order for future growth is:

1. better visual control
2. AI-assisted generation
3. rehearsal and memory practice
4. export and sharing
5. multi-layout support
6. cloud and collaboration
7. advanced input sources
8. personal visual libraries
9. analytics

---

## 13. Summary

The future phases expand ClockRail from a visual memory-mapping MVP into a more intelligent, personalized, and collaborative product. Each phase should be added only after the previous phase is stable enough to support it.
